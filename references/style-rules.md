# 写作风格规范（少数派精简版）

以下规则从少数派写作风格指南中筛选，适用于论坛图文帖。标注「强制」的必须遵守，「推荐」的建议采纳。

## 标点与排版

### 中英文空格（强制）
- 中文与英文/数字之间加一个半角空格：「iPhone 15 Pro」「2026 年 7 月」
- 英文旁有中文标点时，与标点间不加空格

### 引号（强制）
- 中文引用使用直角引号「」，嵌套用『』
- **全文尽量少用引号**，禁止用引号做强调或装饰
- 不用直引号 " 和 '

### 省略号（强制）
- 使用两个半角省略号「……」，不用句号代替

### 连接号
- 中文起止/说明用「—」（一个长连接号）
- 中文说明/中断用「——」（两个长连接号）

### 禁止
- 不用斜线表示并列选项
- 不用间隔号等点状符号标示列表

## 文本样式

### 标题（强制）
- 层级连续不跳跃，不能 H1 直接跳 H3
- 不为了放大字体而用标题样式
- 全文只有一个主标题（H1）

### 粗体（强制）
- 用于关键词强调
- **禁止**：频繁加粗、连续加粗、大面积加粗、用粗体代替标题
- 粗体末尾的标点不加粗（整句加粗除外）

### 段落（推荐）
- 长短适中、逻辑自成一体
- 不出现只有一句话的段落，除非确有必要

### 斜体（强制）
- 任何情况不得将中文设为斜体

### 大小写（强制）
- 中文句内夹用普通英文单词，首字母一律小写
- 专有名词保留官方大小写：GitHub、NVIDIA、Sony、macOS

### 块引用（强制）
- 仅用于实际引用内容，内部不加引号
- **禁止仅为强调或装饰使用块引用**

## 链接

### 链接文本（强制）
- 须能说明链接目标内容
- **禁止**：「点击」「这里」「下载」等无信息量文本做链接
- **禁止**：裸链接地址（除非上下文需要）

### 链接地址
- 去除追踪参数（utm_、fb_、ga_ 等）
- 优先使用固定链接（permalink）

## 专有名词

| 名词 | 正确写法 | 禁止写法 |
|------|---------|----------|
| App | App 或 应用 | APP |
| GitHub | GitHub | Github, Gayhub |
| NVIDIA | NVIDIA | N 厂 |
| Sony | Sony | 大法 |
| macOS | macOS | MacOS, macos |
| Android | Android | 可用「安卓」 |

## 数字与日期

### 日期（强制）
- 完整句子中使用汉字表达：「2026年7月17日」
- 不用横线/斜线格式
- 不用前置零

### 货币（推荐）
- 使用「数值 货币单位」：「100 美元」而非「$100」
- 外币可括号加注人民币换算

## 图片

### 题注（强制）
- 每张图片有题注
- 不通过题注表达正文未包含的信息（除引注来源外）

### 版权（强制）
- 使用他人图片须标注来源
- 照片标注拍摄者

### 遮挡敏感信息（强制）
- 用纯色色块遮挡，**不用马赛克或模糊**（马赛克可被还原）

## 关键词效力

- **强制**：必须遵守
- **推荐**：无充分理由须采纳
- **可以**：作者自行决定
